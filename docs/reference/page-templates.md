# 页面模板

这些模板用于保持 Yetrealm Wiki 的长期一致性。新增页面时优先复制相近模板，再按实际内容删改。

## 物品 / 方块图鉴条目

```markdown
## 分类名称

| 名称 | ID | 获取方式 | 用途 / 备注 |
| --- | --- | --- | --- |
| 物品显示名 | `yetrealm:item_id` | 掉落 / 合成 / 世界生成 | 玩家实际用途、关键限制、关联页面 |
```

写作要求：

- 名称来自 `zh_cn.json`，必要时保留英文名。
- ID 必须可复制，格式为 `yetrealm:*`。
- 如果有图标，使用 `docs/assets/yetrealm/textures/...` 中的实际纹理。
- 不确定数值不要写猜测；先写“来源待核对”并在提交前补证据。

## 生物 / Boss 页面

```markdown
# 生物或 Boss 名称

## 定位

一句话说明它在路线中的作用。

## 出现位置

| 条件 | 说明 |
| --- | --- |
| 维度 / 群系 | 具体位置 |
| 召唤物 / 结构 | 触发条件 |

## 战斗要点

- 准备事项。
- 高风险技能。
- 推荐阶段和装备。

## 掉落与用途

| 掉落 | 用途 |
| --- | --- |
| `yetrealm:item_id` | 解锁材料链或后续 Boss |
```

## 系统机制页

```markdown
# 系统名称

## 一句话说明

这个系统解决什么问题，玩家什么时候会接触到。

## 玩家规则

| 操作 / 条件 | 结果 |
| --- | --- |
| 触发条件 | 实际效果 |

## 配置与服务器风险

| 配置键 | 默认 | 说明 |
| --- | --- | --- |
| config.key | value | 对服务器或整合包的影响 |
```

写作要求：

- 玩家规则使用普通语言，配置表保留真实 key。
- 涉及公共服风险时使用 `!!! warning`。
- 数值来自配置类、语言文件或 Java 常量。

## 攻略页

```markdown
# 攻略主题

## 适用阶段

这个攻略适合新手、中期、终局还是服务器管理。

## 路线

| 阶段 | 地点 | 目标 | 结果 |
| --- | --- | --- | --- |
| 1 | 地点 | 要做什么 | 解锁什么 |

## 准备清单

- 装备。
- 消耗品。
- 风险处理。
```

写作要求：

- 每一阶段都回答“为什么要做这一步”。
- 不把长表格塞进攻略页；长表格放到参考页。

## 参考页

```markdown
# 参考主题

## 数据来源

说明这页主要来自哪些源码目录或 JSON。

## 索引

| ID / Key | 显示名 | 路径 | 说明 |
| --- | --- | --- | --- |
| `yetrealm:*` | 名称 | `data/yetrealm/...` | 用途 |
```

写作要求：

- 参考页可以密集，但必须方便搜索。
- 配方、掉落、配置、命令都要保留真实路径或 key。
