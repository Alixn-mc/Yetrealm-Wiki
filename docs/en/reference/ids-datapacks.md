# IDs and Datapacks

Keep IDs visible in public documentation. They help players search logs, server owners write datapacks, and maintainers verify generated content.

## Common Namespaces

| Namespace | Use |
| --- | --- |
| `yetrealm` | Mod blocks, items, entities, dimensions, recipes, loot, skills, and structures |
| `curios` | Curios slot definitions and item tags |
| `minecraft` | Vanilla tags, recipes, and compatibility entries |

## Datapack Areas

- `data/yetrealm/recipe/`
- `data/yetrealm/loot_table/`
- `data/yetrealm/worldgen/`
- `data/yetrealm/skills/skill_tree.json`
- `data/curios/curios/items/`
- `data/yetrealm/tags/`

Avoid changing public IDs unless a release explicitly documents migration behavior.
